# Edge cases — analyze-demand

## Two-sided platforms
Run JTBD analysis on EACH side separately (consumer side AND merchant side, rider AND driver, advertiser AND audience). The two sides almost always have different substitution risks and different WTP drivers. A single combined analysis hides the structural risk.

## Commodity / spec-defined industries
JTBD is degenerate ("provide the spec'd product at the agreed price"). State this explicitly, weight analysis toward substitution + WTP drivers only, do not force-fit emotional/social jobs. The validator still requires the three components named; state "Emotional/social: minimal — buyers select on price + lead-time only" rather than omitting.

## Regulated markets where buyer choice is constrained
JTBD is real but secondary to regulatory structure. Defense procurement, regulated utilities, FDA-restricted pharma channels. State the regulatory constraint up-front; the demand analysis is then constrained-choice rather than free-choice.

## New / pre-product-market-fit categories
The most important segment is often "do-nothing" — the buyer continues to live with the problem because no available solution is worth the switching cost. Force the analysis to assess do-nothing as a segment, not just as a substitute.

## Industries in active disruption
When a cross-category substitute is already taking visible share (e.g., GLP-1 vs diet programs as of 2024-26), the JTBD analysis must call this out as the PRIMARY structural finding, even if traditional metrics still look stable. The point of the skill is to catch this BEFORE share data moves; if share is already moving, the analysis is late but should still name the shift clearly.

## Buying-committee asymmetry (B2B)
When one stakeholder JTBD dominates ("CFO career risk on this category is so high that it determines all purchases regardless of user preference"), state this explicitly. The binding constraint is the analytical answer.

## Markets with extreme demographic correlation
Occasionally a JTBD IS strongly correlated with a demographic (e.g., menopause hormone therapy buyers are demographically defined by age + sex + life-stage). State the correlation, lead the segment name with the JOB clause anyway, and explain why the demographic is causally tied rather than decorative.

## Skip-condition edge case
If the orchestrator skipped this analysis on the mature-stable rule but the user runs it standalone anyway, the analysis often surfaces a substitution risk the Five Forces missed. This is the entire reason for the conditional rule rather than the hard-skip rule.
